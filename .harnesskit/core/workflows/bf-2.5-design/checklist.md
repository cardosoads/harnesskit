# Validation Checklist: Design Audit

---

## Required Items

- [ ] **Current state documented** — Is the existing design state mapped?
  - Criterion: at least colors, typography, and component inventory documented.

- [ ] **Gaps identified** — Are inconsistencies and problems listed?
  - Criterion: at least 3 gaps identified with severity and impact.

- [ ] **Design tokens proposed** — Is a token system proposed to unify patterns?
  - Criterion: proposed tokens cover at least colors, typography, and spacing.

- [ ] **Improvement plan exists** — Is there a prioritized plan?
  - Criterion: at least 3 improvements listed with priority and effort.

---

## Optional Items (recommended)

- [ ] **Accessibility assessed** — Is the current a11y state documented?
  - Criterion: contrast, keyboard nav, and semantic HTML assessed.

- [ ] **Migration strategy defined** — Is there a path from current to proposed?
  - Criterion: at least a high-level migration approach described.

---

## Validation Result

- **All required items approved?** -> Proceed to implementation.
- **Any required item failed?** -> Fix before proceeding.
