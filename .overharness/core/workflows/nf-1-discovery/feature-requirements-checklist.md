# Validation Checklist: Feature Requirements

Validate each item below against the generated report.

---

## Required Items

- [ ] **Every feature has functional requirements** — Does each feature have at least 2 functional requirements?
  - Criterion: every feature section contains a functional requirements list with 2+ items.

- [ ] **Every feature has acceptance criteria** — Does each feature have at least 2 acceptance criteria?
  - Criterion: every feature section contains acceptance criteria in Given/When/Then format.

- [ ] **Impact analysis referenced** — Does the document reference Raj's impact analysis findings?
  - Criterion: at least one feature references the impact summary from Raj.

- [ ] **Scope defined** — Are in-scope and out-of-scope clearly listed?
  - Criterion: the scope section has at least one item in each category.

- [ ] **Summary matrix present** — Is there a summary table with all features at a glance?
  - Criterion: matrix includes priority, effort, risk, and counts.

---

## Optional Items (recommended)

- [ ] **Non-functional requirements present** — Are there non-functional requirements for at least one feature?
  - Criterion: at least one feature has NFRs documented.

- [ ] **Business rules documented** — Are business rules captured for features that have them?
  - Criterion: at least one feature has business rules.

- [ ] **Scope changes documented** — If scope changed from the original proposal, is it noted?
  - Criterion: changes section exists if the user modified scope during requirements.

---

## Validation Result

- **All required items approved?** -> Report approved. Proceed to specification.
- **Any required item failed?** -> Go back to fix it.
- **Optional items failed?** -> Inform but do not block.
